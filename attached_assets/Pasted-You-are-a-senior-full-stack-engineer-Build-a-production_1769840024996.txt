You are a senior full-stack engineer. Build a production-ready, multi-tenant landing page + admin panel platform to run on Replit.

PROJECT NAME
- "university-landing-platform"

PRIMARY GOAL
- We will host the first tenant at: okanuniversity.app
- This platform is for paid-ads traffic: high conversion, fast, technically SEO-correct, and easy to customize per university via an admin panel.
- The main site already exists elsewhere; this app is purely conversion landing + application intake + automations.

LANGUAGES / CONTENT
- Default site language: English (en)
- Also support: Arabic (ar), Turkish (tr), French (fr), Russian (ru), Farsi (fa)
- Admin must be able to enter content for each language.
- Public landing must have a language switcher.
- If a piece of content is missing in a language, fallback to English.
- RTL must be supported for Arabic + Farsi (html dir="rtl" + proper UI alignment).

MUST-HAVE FEATURES (PUBLIC)
1) Landing Page
- WhatsApp-style minimal conversion layout:
  - Hero section with university name, main promise, CTA
  - Trust badges section
  - Program Finder section with filters
  - "3 steps" application process section
  - Testimonials section (optional)
  - FAQ section (optional)
  - Contact / Footer
- ALL of the above sections must be togglable (visible on/off) from admin.
- Menu structure (which sections appear + order + labels) must be controlled by admin.

2) Program Finder
- Student can filter and select programs.
- Programs fields:
  - University
  - Program Name
  - Degree
  - Language
  - Tuition Fee
  - Our Discounted Fee
  - (optional but recommended) external_program_id (for mapping to portal)
- Filters: Degree, Language, Fee range, search by name.
- Program selection must start the application flow without creating SEO-bloating query-param URLs.
- Use clean routes like:
/[lang]/apply -> application wizard
/[lang]/programs -> optional listing (can be embedded on landing)

3) Application Flow (Wizard)
- Step 1: Lead capture (Full name, phone with country code, email)
- Step 2: Program confirmation (selected program)
- Step 3: Document upload + AI checks + autofill
- Step 4: Review & Submit
- Enforce ALL user-entered text data to be ENGLISH UPPERCASE:
  - On frontend: auto-transform to uppercase (onChange)
  - On backend: validate and reject lowercase or transform before persistence
- When application is submitted:
  - Store application + docs metadata
  - Trigger outbound webhook to portal.findandstudy.com (see Integration section)
  - Send confirmation email to the student (multi-language template)

4) Chatbot Widget (WhatsApp look)
- A floating chat button (bottom corner), openable into WhatsApp-style chat UI.
- Admin can enable/disable chatbot per tenant.
- Admin can set welcome message per language.
- Messages are sent to backend endpoint; backend forwards to n8n webhook and returns AI response.
- Include support for:
  - session_id
  - tenant_id
  - language
  - selected_program_id (if any)
- The widget must look like WhatsApp: message bubbles, timestamps, typing indicator, input bar.

INTEGRATION: PORTAL.FINDANDSTUDY.COM (CRITICAL)
- On application submission, POST data to portal.findandstudy.com webhook.
- Implement robust delivery using an Outbox table + worker:
  - Never lose events
  - Retry with exponential backoff
  - Idempotency key to avoid duplicates
- Security:
  - HMAC-SHA256 signature header over raw body using shared secret
  - Timestamp header
  - Idempotency key header
- Payload must include:
  - tenant domain and university name
  - application_id, status, submitted_at
  - applicant (UPPERCASE)
  - program info (UPPERCASE + fees)
  - documents array with secure file URLs (do NOT send large binaries)
  - utm parameters (if available)
- Provide a “Test webhook” button in admin.

DOCUMENT UPLOAD + AI CHECKS (MVP + EXTENSIBLE)
- Student uploads documents (passport, diploma, transcript, etc.).
- Store files in a simple storage:
  - Prefer S3-compatible (if available) but include local disk storage fallback in Replit.
- After upload:
  - Run checks:
    - expiry date validity (passport)
    - name match between documents and form
    - age check from birth date
    - grade/eligibility check (basic rules)
  - Extract fields to auto-fill missing form fields.
- Because OCR/AI providers vary, build it as a pluggable pipeline:
  - interface: DocumentProcessor
  - default implementation: stub/mock extraction + simple regex parsing
  - optional integration: call an external AI/OCR webhook if configured
- Save extracted fields and checks into DB.
